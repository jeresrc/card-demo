# Cycle card between three color variants on click/tap

## Goal

Cycle card through three variants on click: `AMAR 100` (purple) → `AMAR 200` (lavender) → `AMAR PREMEDIC` (teal) → back. Each transition is a quick ~200ms snap where all color values actually animate (not just swap). Every variant has:
- a gradient stroke (vertical, FFFFFF → variant accent, 15% opacity, inside, 3px/4px),
- a film-grain noise overlay covering the **whole card** (not just the stroke).

## Incremental update — add full-card noise + gradient stroke per variant

### Figma specs

| Variant | Stop 0% | Stop 100% | Weight | Opacity | Direction |
|---|---|---|---|---|---|
| AMAR 100 | `#FFFFFF` | `#7C3AED` | 3px | 15% | vertical (FFF top → color bottom) |
| AMAR 200 | `#FFFFFF` | `#7C3AED` | 3px | 15% | vertical |
| AMAR PREMEDIC | `#FFFFFF` | `#42EDEA` | 4px | 15% | vertical |

Stroke `Position: Inside` → render the ring inside the rounded card without affecting layout. The Figma `Effects: Noise` on the stroke is part of a wider design intent: **the entire card has a sutle film-grain texture**, so we apply the noise as a full-card overlay (not only on the ring). The ring just gets the gradient.

Add **two new overlay divs as the last children of `<article>`** (after the sheen band, so they paint on top of all the holo layers):

```tsx
{/* Gradient border (vertical FFF → variant accent), 15% opacity */}
<div
  aria-hidden
  className="pointer-events-none absolute inset-0 rounded-[24px]"
  style={{
    padding: 'var(--card-border-w)',
    background:
      'linear-gradient(180deg, #ffffff 0%, var(--card-border-c) 100%)',
    opacity: 0.15,
    WebkitMask:
      'linear-gradient(#000 0 0) content-box, linear-gradient(#000 0 0)',
    WebkitMaskComposite: 'xor',
    mask: 'linear-gradient(#000 0 0) content-box, linear-gradient(#000 0 0)',
    maskComposite: 'exclude',
  }}
/>

{/* Full-card film-grain noise */}
<div
  aria-hidden
  className="pointer-events-none absolute inset-0 rounded-[24px]"
  style={{
    backgroundImage: NOISE_URL,
    backgroundSize: '160px 160px',
    backgroundRepeat: 'repeat',
    mixBlendMode: 'overlay',
    opacity: 0.18,
  }}
/>
```

How each works:
- **Gradient border**: `padding: var(--card-border-w)` reserves a ring between border-box and content-box. Two stacked masks combined via `mask-composite: exclude` cut out the inner rectangle, leaving only the ring painted with the FFF→accent gradient at 15% opacity.
- **Full-card noise**: a small tileable SVG noise (`feTurbulence`) repeated across the whole card area at `mixBlendMode: 'overlay'` and `opacity: 0.18` for sutle film-grain. `rounded-[24px]` keeps it clipped to the card's silhouette (combined with the parent's `overflow-hidden`).
- Both have `pointer-events: none` so clicks still hit the article underneath.
- The noise opacity is intentionally fixed (not driven by variant) — same grain density on every variant.

No other code paths change. The cycle, the holo overlays, the heart-cat SVG, the pointer-tilt RAF — all untouched.

### Notes / caveats (gradient border)

- `<length>` registered properties animate identically to `<color>`, so `--card-border-w` going `3px ↔ 4px` will smoothly interpolate within the 200ms snap.
- Browser support for `mask-composite: exclude` is broad in 2025 (Chromium 120+, Safari 15.4+, Firefox 53+). On unsupported engines the ring degrades to a filled rounded rectangle with 15% opacity — visually similar enough.
- The noise tile is 120×120; tile size doesn't have to match the ring width since the mask clips it.

### Verification (gradient border + noise)

1. The whole card surface has a sutle film-grain texture, visible against the gradient background.
2. Each variant shows a faint white→accent vertical gradient ring inside the rounded corners.
3. AMAR PREMEDIC's ring is visibly thicker than the purple variants' (4px vs 3px).
4. Ring color + width transitions smoothly during the variant cycle (no snap).
5. Clicks on the ring/noise area still trigger `cycleVariant` (because both overlays are `pointer-events: none`).
6. The grain is rendered crisp, not pixelated — at typical card sizes the 160px noise tile reads as proper film grain.

## Approach

## Notes / caveats

- `@property` is supported in all modern Chromium/WebKit/Firefox (Firefox since 128, March 2024). Without it the transition silently snaps — acceptable degradation given target is a portfolio demo.
- The pointer-tilt RAF loop still writes `--mx`, `--my`, `--fx`, `--fy`, `--hi`, `--tx`, `--ty`; those remain unregistered (no transition) so tilt stays buttery-smooth.
- Cycling on `onClick` of `<article>` won't conflict with `pointermove` tilt (clicks fire on `mouseup` regardless). No drag-threshold logic needed per user choice.
- The peachy bloom in variant 2 is part of the base `background` (radial layer with animated `--card-bloom` color from transparent → peach), so it persists statically and isn't tied to pointer proximity.
