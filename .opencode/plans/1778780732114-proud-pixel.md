# Plan: Interactive Tilt + Reflective Light (Phase 2)

## Goal
Make the card feel alive: subtle 3D tilt driven by **mouse** (desktop) or **gyroscope** (mobile), plus a **soft moving highlight** that tracks the same input, simulating a glossy reflective surface.

## Scope
Single file change: `src/routes/index.tsx`. No new dependencies. Plain React hooks + CSS custom properties. Respect `prefers-reduced-motion`. SSR‑safe (all sensors accessed inside `useEffect`).

## Files to Modify
- `src/routes/index.tsx` — extract card into a small client component with effects + add a highlight overlay layer.

## Design Spec

### Motion (tilt)
- Base rotation stays `-4deg` (the resting pose).
- Add `rotateX` and `rotateY` driven by input, clamped to **±6°** total range.
- Wrap the card in a perspective container: `perspective: 1000px`.
- Final transform: `rotate(-4deg) rotateX(var(--ty)) rotateY(var(--tx))` on a `translateZ(0)` layer for GPU.
- Animation is smoothed: every rAF tick the current values lerp toward the target (`current += (target - current) * 0.12`). This keeps the look "muy sutil" and prevents jitter from sensor noise.

### Light / Reflex
- Add an absolutely positioned, full‑bleed `<div>` inside the card (above content but `pointer-events-none`) using:
  ```css
  background: radial-gradient(
    420px circle at calc(var(--mx) * 1%) calc(var(--my) * 1%),
    rgba(255, 255, 255, 0.45),
    rgba(255, 255, 255, 0.12) 18%,
    transparent 45%
  );
  mix-blend-mode: soft-light;
  ```
- Second thinner sheen band for the "media reflex" look — a diagonal `linear-gradient` overlay whose offset is bound to `--mx` (translates a thin white strip across the card), `mix-blend-mode: overlay`, opacity ~0.25.
- Both overlays inherit the card's `border-radius` and `overflow: hidden` clipping.

### Inputs
1. **Pointer (desktop / pen / touch‑drag)**
   - `pointermove` on the **card stage** (the centered container) — use stage rect so the effect tracks even outside the card.
   - Map pointer position → `mx`, `my` in `[0,100]` and `tx`, `ty` in `[-6,6]`.
   - On `pointerleave` lerp back to center (50, 50, 0, 0).

2. **Gyroscope (mobile)**
   - Listen to `deviceorientation`. Use `gamma` (left/right tilt, −90…90) → `tx`; `beta` (front/back, −180…180, clamped to −45…45) → `ty`.
   - Scale: divide by 8 then clamp to ±6°. Map the same values to `mx`/`my` around 50%.
   - **iOS gate**: if `typeof DeviceOrientationEvent.requestPermission === 'function'`, render a small "Activar movimiento" pill button overlaid on first load. Tapping it calls `requestPermission()`; on `'granted'` we attach the listener and hide the pill. Android/desktop browsers that don't need permission attach immediately.
   - When gyro is active, pointer input is ignored to avoid conflicts.

3. **Reduced motion**
   - If `matchMedia('(prefers-reduced-motion: reduce)').matches`, skip all listeners and rAF. Card stays static. Light overlay stays at default center.

### State / Variables
Card uses CSS custom properties set via `style`:
- `--mx`, `--my`: 0..100 (light position, default 50/50)
- `--tx`, `--ty`: degrees (tilt, default 0/0)

A single `useEffect` owns:
- A `targetRef` (object with mx,my,tx,ty) updated by event handlers.
- A `currentRef` (same shape) lerped each rAF tick.
- Writes the lerped values to the card element's `style.setProperty` directly (no React re-renders).

## Implementation Sketch

```tsx
function Card() {
  const stageRef = useRef<HTMLDivElement>(null)
  const cardRef = useRef<HTMLElement>(null)
  const [needsPermission, setNeedsPermission] = useState(false)

  useEffect(() => {
    const reduce = matchMedia('(prefers-reduced-motion: reduce)').matches
    if (reduce) return

    const target = { mx: 50, my: 50, tx: 0, ty: 0 }
    const current = { ...target }
    let raf = 0
    let usingGyro = false

    const apply = () => {
      const c = cardRef.current
      if (c) {
        c.style.setProperty('--mx', current.mx.toFixed(2))
        c.style.setProperty('--my', current.my.toFixed(2))
        c.style.setProperty('--tx', current.tx.toFixed(2) + 'deg')
        c.style.setProperty('--ty', current.ty.toFixed(2) + 'deg')
      }
    }
    const loop = () => {
      const k = 0.12
      current.mx += (target.mx - current.mx) * k
      current.my += (target.my - current.my) * k
      current.tx += (target.tx - current.tx) * k
      current.ty += (target.ty - current.ty) * k
      apply()
      raf = requestAnimationFrame(loop)
    }
    raf = requestAnimationFrame(loop)

    const onPointerMove = (e: PointerEvent) => {
      if (usingGyro) return
      const r = stageRef.current!.getBoundingClientRect()
      const nx = (e.clientX - r.left) / r.width   // 0..1
      const ny = (e.clientY - r.top) / r.height
      target.mx = clamp(nx * 100, 0, 100)
      target.my = clamp(ny * 100, 0, 100)
      target.tx = (nx - 0.5) * 12   // ±6deg
      target.ty = -(ny - 0.5) * 12
    }
    const onPointerLeave = () => {
      target.mx = 50; target.my = 50; target.tx = 0; target.ty = 0
    }

    const onOrient = (e: DeviceOrientationEvent) => {
      usingGyro = true
      const gx = clamp((e.gamma ?? 0) / 6, -6, 6)
      const gy = clamp((e.beta ?? 0) / 8, -6, 6)
      target.tx = gx
      target.ty = -gy
      target.mx = clamp(50 + gx * 6, 0, 100)
      target.my = clamp(50 + gy * 6, 0, 100)
    }

    const stage = stageRef.current!
    stage.addEventListener('pointermove', onPointerMove)
    stage.addEventListener('pointerleave', onPointerLeave)

    const DOE = (window as any).DeviceOrientationEvent
    if (DOE && typeof DOE.requestPermission === 'function') {
      setNeedsPermission(true)
    } else if (DOE) {
      window.addEventListener('deviceorientation', onOrient)
    }

    // Expose attach for the permission button
    ;(window as any).__amarAttachGyro = async () => {
      try {
        const res = await DOE.requestPermission()
        if (res === 'granted') {
          window.addEventListener('deviceorientation', onOrient)
          setNeedsPermission(false)
        }
      } catch {}
    }

    return () => {
      cancelAnimationFrame(raf)
      stage.removeEventListener('pointermove', onPointerMove)
      stage.removeEventListener('pointerleave', onPointerLeave)
      window.removeEventListener('deviceorientation', onOrient)
      delete (window as any).__amarAttachGyro
    }
  }, [])

  return (
    <div ref={stageRef} className="grid min-h-dvh place-items-center p-8" style={{ perspective: '1000px' }}>
      <article
        ref={cardRef}
        className="relative isolate aspect-[7/9] w-[min(440px,88vw)] overflow-hidden rounded-[28px] px-10 py-9 text-white will-change-transform"
        style={{
          background: 'linear-gradient(160deg, #7558ad 0%, #694C9A 55%, #5a3f88 100%)',
          boxShadow: '0 30px 60px -20px rgba(60, 40, 100, 0.45), 0 12px 24px -10px rgba(0,0,0,0.25)',
          transform: 'rotate(-4deg) rotateX(var(--ty, 0deg)) rotateY(var(--tx, 0deg))',
          transformStyle: 'preserve-3d',
        }}
      >
        {/* …existing content (kicker, name, id, deco svg)… */}

        {/* Reflective highlight */}
        <div
          aria-hidden
          className="pointer-events-none absolute inset-0"
          style={{
            background:
              'radial-gradient(420px circle at calc(var(--mx,50) * 1%) calc(var(--my,50) * 1%), rgba(255,255,255,0.45), rgba(255,255,255,0.12) 18%, transparent 45%)',
            mixBlendMode: 'soft-light',
          }}
        />
        {/* Diagonal sheen band */}
        <div
          aria-hidden
          className="pointer-events-none absolute inset-0 opacity-25"
          style={{
            background:
              'linear-gradient(115deg, transparent 40%, rgba(255,255,255,0.55) 50%, transparent 60%)',
            transform: 'translateX(calc((var(--mx,50) - 50) * 1.2%))',
            mixBlendMode: 'overlay',
          }}
        />
      </article>

      {needsPermission && (
        <button
          type="button"
          onClick={() => (window as any).__amarAttachGyro?.()}
          className="fixed bottom-6 left-1/2 -translate-x-1/2 rounded-full bg-white/90 px-4 py-2 text-sm font-semibold text-[#5a3f88] shadow-lg backdrop-blur"
        >
          Activar movimiento
        </button>
      )}
    </div>
  )
}
```

Notes:
- `clamp(n,a,b)` is a tiny local helper.
- Card content z‑order: SVG decoration stays below the two overlay layers so the light passes over it too. The text remains above thanks to `position: relative` from utilities (kicker/name are static flow above the absolute overlays, but overlays are absolute and rendered later in DOM — text appears underneath. Fix: give text wrappers `relative z-10` so they sit above the highlight; light then bathes the background and SVG only.)

# Plan: Pink/Violet Bloom Reflex on SVG (Phase 4 — supersedes Phase 3 reflex)

## Goal
Replace the current rainbow + `difference` reflex (which renders brown/muddy on purple) with an **Airbnb-style pink → magenta → violet → coral bloom**. When the light passes the heart‑cat region, the silhouette and a soft halo around it intensify into a vivid warm‑pink wash, just like the reference screenshot.

## Diagnosis
- `mix-blend-mode: difference` between a rainbow and a violet card subtracts channels → low-chroma brown/olive output.
- Airbnb's effect is **additive / tinting**, not subtractive: a saturated pink+violet light dodges/screens onto the surface, brightening and saturating the underlying graphic.

## Approach (3 layers, same masking strategy as before)

All three layers are positioned `absolute inset-0` inside the card, with the existing `--mx`, `--my` driving position. The SVG silhouette mask sits at `right -8px bottom -12px / 170px auto`. Layers in DOM order (lowest → highest):

1. **Pink halo (unmasked, soft)** — gives the "the whole corner is glowing" feel.
   - Background: radial-gradient at `(--mx%, --my%)` with stops:
     `rgba(255,90,160,0.55) 0%, rgba(200,60,255,0.35) 18%, transparent 45%`.
   - No silhouette mask (only radial intensity).
   - `mix-blend-mode: screen`.
   - `opacity: 0.85`.

2. **Saturated tint (masked to silhouette + light spot)** — paints the heart/cat strongly pink-violet-coral where the light is.
   - Background: conic-gradient kept in warm palette only (rotates subtly with `--mx`):
     ```
     conic-gradient(from calc(var(--mx,50) * 1.8deg) at 50% 50%,
       #ff3d8b, #ff5677, #ff8a4c, #c84cff, #a836ff, #ff3d8b)
     ```
     (no greens/blues — only pinks, magentas, violets, coral).
   - Mask: silhouette **+** radial spotlight at `(--mx%, --my%)` (black 0–14%, transparent 32%), `mask-composite: intersect`.
   - `mix-blend-mode: plus-lighter` (additive — boosts saturation toward overlay).
   - `opacity: 0.95`.

3. **Bright specular (masked to silhouette)** — small near-white core for the "wet/sticker" highlight.
   - Background: `radial-gradient at (--mx%, --my%)` with `rgba(255,220,235,0.95) 0%, rgba(255,150,200,0.4) 10%, transparent 22%`.
   - Mask: silhouette only.
   - `mix-blend-mode: color-dodge` (intensifies where pink already exists; clamps to white otherwise).
   - `opacity: 0.8`.

### Why the colors come through
- `plus-lighter` and `screen` are additive: applied over a violet (#694C9A) surface, a vivid pink (#ff3d8b) overlay produces a hot magenta — never brown.
- `color-dodge` further blows out the brightest areas, mimicking Airbnb's sun‑lit "verified" pop.
- Limiting the palette to warm/violet hues guarantees no muddy intermediate tones.

## Removed
- The previous holographic `difference` overlay and its conic rainbow palette (`#ffd25c`, `#5cff9d`, `#5cd6ff`, etc.). Replaced wholesale.

## Files to Modify
- `src/routes/index.tsx` — replace the two existing reflex `<div>`s (the conic+difference and screen specular) with the three new layers described above. Keep `HEART_CAT_MASK_URL` constant unchanged.

## Verification

2. Expected: the silhouette region floods with hot pink / magenta / violet (no brown, no rainbow). A soft pink halo extends just beyond the silhouette into the card.
3. Move cursor to the top‑left: bloom fades to almost nothing in the SVG region; the corner stays purple.
4. `pnpm exec tsc --noEmit` clean.

## Out of Scope
- Changing the card base color, the existing white soft‑light highlight, or the diagonal sheen band — they stay.
- Idle animation when the cursor is stationary.

# Plan: Stronger Tilt + Holographic Reflex on SVG (Phase 3 — superseded by Phase 4)

## Goal
1. Make the rotation noticeably more pronounced.
2. When the light passes over the decorative heart‑cat SVG, paint a moving **rainbow / negative‑color reflex** on the silhouette — like a holographic foil sticker catching the light.

## Scope
Single file: `src/routes/index.tsx`. No new deps. Pure CSS conic gradient + double mask + `mix-blend-mode: difference`. The effect re‑uses the existing `--mx`, `--my` custom properties.

## Changes

### 1. Tilt range
- `MAX_TILT`: `10` → **`16`** (in `onPointerMove`).
- Gyro mapping bumped to `±16` (`/ 2.5` and `/ 3.5` divisors, clamped to ±16).

### 2. Holographic reflex layer

#### Approach
A new `<div>` sits *inside* the card, positioned identically to the `<svg>` decoration (bottom‑right). It carries:

- A **conic rainbow gradient** that rotates with the cursor (`from calc(var(--mx) * 3.6deg)`).
- A **double CSS mask** combined with `mask-composite: intersect`:
  1. Mask A: the SVG silhouette (as a data‑URL `mask-image`) — limits the rainbow to the cat/heart shape.
  2. Mask B: a radial gradient at `(--mx%, --my%)` — limits the rainbow to the area currently under the light.
- `mix-blend-mode: difference` — produces the negative‑color "oil slick" reflex when the rainbow lands on the purple background of the SVG region.
- A second `<div>` on top: a small **specular core** (white radial, `mix-blend-mode: screen`) at the same masked region, so the reflex has a brighter hot spot.

#### Silhouette mask
Define once at the top of the file as a constant:
```ts
const HEART_CAT_MASK =
  "url(\"data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 138 169' fill='white'><path d='…heart outline path d…'/><path d='…cat path d…'/></svg>\") center / contain no-repeat"
```
The two `d` attributes come from the existing SVG (paths 1 and 4 — the heart envelope and the cat body). Strokes/inner shadows from the existing SVG are dropped from the mask; we want a solid silhouette so the mask is binary.

#### Overlay markup (inside `<article>`, after the existing `<svg>`, before the two existing light overlays)
```tsx
{/* Holographic reflex — only paints where the light overlaps the SVG silhouette */}
<div
  aria-hidden
  className="pointer-events-none absolute -bottom-3 -right-2 aspect-[138/169] w-[170px]"
  style={{
    background:
      'conic-gradient(from calc(var(--mx, 50) * 3.6deg) at 50% 50%,' +
      ' #ff5ce0, #ffd25c, #5cff9d, #5cd6ff, #b65cff, #ff5c8a, #ff5ce0)',
    WebkitMaskImage: `${HEART_CAT_MASK}, radial-gradient(circle at calc((var(--mx,50) - 65) * 6.6%) calc((var(--my,50) - 65) * 6.6%), #000 0%, #000 14%, transparent 36%)`,
    maskImage: `${HEART_CAT_MASK}, radial-gradient(circle at calc((var(--mx,50) - 65) * 6.6%) calc((var(--my,50) - 65) * 6.6%), #000 0%, #000 14%, transparent 36%)`,
    WebkitMaskComposite: 'source-in',
    maskComposite: 'intersect',
    mixBlendMode: 'difference',
    opacity: 0.85,
  }}
/>
{/* Specular bright spot inside the same silhouette */}
<div
  aria-hidden
  className="pointer-events-none absolute -bottom-3 -right-2 aspect-[138/169] w-[170px]"
  style={{
    background:
      'radial-gradient(circle at calc((var(--mx,50) - 65) * 6.6%) calc((var(--my,50) - 65) * 6.6%), rgba(255,255,255,0.9), rgba(255,255,255,0.2) 18%, transparent 36%)',
    WebkitMaskImage: HEART_CAT_MASK,
    maskImage: HEART_CAT_MASK,
    mixBlendMode: 'screen',
    opacity: 0.75,
  }}
/>
```

Notes on the math:
- The SVG sits at the bottom‑right of the card. Mapping the light `(--mx, --my)` (0..100 of the *card*) into the SVG's local box means shifting the origin: the SVG starts roughly where `mx ≈ 65, my ≈ 65`. We approximate with the linear transform `(mx - 65) * 6.6%` (so when light is centered on the SVG, the inner radial is centered too). Exact numbers may need a small tweak when wired up.

### 3. Why this gives a "rainbow negative reflex"
- The conic gradient supplies the hue cycle.
- `mix-blend-mode: difference` inverts the underlying colors where the rainbow overlay is bright → on the violet card background you get cyan/yellow/green oil‑slick hues instead of pastels; pure negative‑color feel.
- The double‑mask intersection means it never bleeds outside the SVG silhouette, and only where the light is — so when the cursor is far from the SVG corner, the reflex is hidden.

### 4. Reduced motion
Already handled — the rAF loop is skipped under `prefers-reduced-motion`, so `--mx`/`--my` stay at 50/50 and the reflex sits idle in the SVG center at low intensity. No extra work needed.
